Arc API Documentation
The Arc API provides an interface for other programs to interact with the data managed by Arc.

Getting Started
Once the Arc API has been installed, users can programmatically access it through its Component Object Model (COM) interfaces.
IArcApi represents the main entry point into the API and is initially accessed through the creation of its implementation class
ArcApi . Once a user is done with their API session, they should call IArcApi.Dispose().
The following example provides a basic demonstration using PowerShell (x86) to create a new Arc API session, log in, retrieve
record information, and then dispose of the session:
$api = New-Object -comobject Arc.Api.ArcApi
$api.Login("admin", "admin")
$records = $api.GetRecords()
foreach ($record in $records) {
$dateRecorded = $record.DateRecorded
$lastNameKey = $record.Patient.DefaultFieldDefinitionKeys.LastName
$patientLastName = $record.Patient.Fields.GetField($lastNameKey).Value.DisplayText
Write-Host "[$dateRecorded] $patientLastName"
}
$api.Dispose()

For a detailed set of examples using C++, see the C++ sample app that gets installed with the Arc API.
If you are developing a .NET app, you will need to reference the installed Arc.Api.dll assembly directly instead of the registered
COM interfaces.

Legacy API
Backwards-compatibility is supported for legacy FlexComApi users through the IComFlexRecord interface. As a convenience,
IComFlexRecord2 was introduced to provide easy access to the newer IRecord information via its IComFlexRecord2.Record
property. It is recommended that future API usage convert entirely to using IArcApi.
